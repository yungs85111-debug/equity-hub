# Investment Analysis Skills — UI
# 주식 기본적 분석 게임형 대시보드 — UI·교육 콘텐츠·인사이트 규칙

**버전:** 4.4.1-ui (v4.4.0 기반 / pick_today_stock 쿼리 주석 수정)
**포함 섹션:** §5 대시보드 구조 · §6 교육 콘텐츠 사전 · §7 인사이트 자동 생성 · §8 오늘의 종목 · §11 면책 표시 · §12 컬러 시스템
**데이터·계산 규칙:** Skills_core.md 참조
**컨셉:** 뇌동매매 하지 말고, 알고 투자하자
**데이터 원천:** SEC EDGAR (data.sec.gov) — API 키 불필요, 무료
**분석 프레임워크:** 6대 코어 스탯 체계 (수익성 / 성장성 / 안정성 / 효율성 / 밸류에이션 / 현금창출력)
**섹터 분류:** GICS 11개 섹터 (SIC → GICS 자동 변환)
**UI 방향:** 게임 스탯 트리 — 섹터 허브 → 기업 비교 → 기업 상세 드릴다운
**상태 표현:** S / A / B / C 티어 + 스테이터스 레이블 병행
**교육 목적:** 복잡한 재무지표를 숫자 나열이 아닌 상태판·속성치·티어·비교 우위로 변환해 누구나 빠르게 해석할 수 있도록 한다.
**타깃 사용자:** 투자를 시작한 지 1~3년 차, 재무제표를 읽고 싶지만 어디서 시작해야 할지 모르는 30~40대. 게임 메타포(S/A/B/C 티어, 캐릭터 시트)는 시각적 이해 도구이며, 투자 전문가용 분석 도구가 아님을 모든 화면에서 명확히 한다.

## 0. UI 공통 표시 규칙 (v4.4.0 추가)

### 데이터 기준일 표시 (데모 모드 투명성)

```
모든 화면 상단 고정 배너:
┌─────────────────────────────────────────────────┐
│ 📅 데이터 기준: {price_date}  |  출처: SEC EDGAR │
│ 데모 버전 — 실시간 데이터가 아닙니다            │
└─────────────────────────────────────────────────┘

표시 조건: DATA_MODE = "demo" 일 때 항상 표시
색상: 카드 배경(#0f172a), 텍스트 #94A3B8 (보조 텍스트)
→ 눈에 거슬리지 않되, 심사위원이 데이터 기준을 확인할 수 있는 수준
```

### 벤치마크 출처 표시

```
섹터 정상 범위를 표시하는 모든 위치에 출처를 함께 표기한다.

동적 계산 시 (§4.0 분위수):
  "섹터 중앙값 {p50} ({source})"
  예시: "섹터 중앙값 18.3% (demo.db 실측 10개사, 2023년 기준)"

Fallback 사용 시 (§4.1 절대값):
  "⚠️ 섹터 데이터 부족 — S&P 500 평균 기준 적용"
```

### 게임 메타포 사용 원칙

```
✅ 사용: S/A/B/C 티어 라벨, 바 게이지, 캐릭터 시트 레이아웃
✅ 사용: "이 섹터를 탐험하세요" 같은 가벼운 표현

❌ 금지: "팩션 선택", "클래스 선택" 등 게임 전용 용어
         → 투자 비전문가가 맥락 없이 접하면 혼란
❌ 금지: 게임 전투/승패 메타포 (재무 분석과 괴리)

원칙: 게임 언어는 숫자를 빠르게 읽기 위한 장치.
      "투자 공부 앱"이라는 첫인상을 해치지 않는다.
```

## 5. 게임형 대시보드 구조

> 기업 단독 뷰보다 **섹터 우선 탐색 → 기업 드릴다운** 구조가 기본 아키텍처다.
> 집단 메타에서 개별 유닛으로 내려가는 전략게임 구조와 동일하다.

> ⚠️ **§5 전체 공통 규칙:** 아래 모든 레이아웃 박스는 예시 구조다.
> `{변수}` 형식의 값은 **하드코딩 금지** — 반드시 런타임에 DB에서 조회한 실제 값으로 채운다.

### 5.1 전체 정보 흐름

```
[홈] 섹터 허브
  ↓ 섹터 선택
[섹터 스탯 보드] 섹터 평균 · 분위수 · 대표 지표
  ↓ 기업 탐색
[기업 비교 화면] 시총 상위 10~20종목 랭킹 테이블
  ↓ 기업 선택
[기업 상세] 캐릭터 시트 — 6대 코어 스탯 트리
  ↓ 지표 클릭
[지표 설명 팝업] 정의 · 섹터 기준 · 현재 상태 · 학습 포인트
```

### 5.2 홈 — 섹터 허브

**디자인 모티브:** 카드 선택형 탐색 UI (게임 팩션/클래스 선택 스타일 — 용어는 투자 맥락으로 표현)

**그리드 배치 규칙 (섹터 11개):**
```
3열 × 4행 그리드 사용. 마지막 행 3번째 셀은 빈 칸 또는 "전체 보기" 카드로 채운다.
행 1: IT / 금융 / 헬스케어
행 2: 경기소비재 / 필수소비재 / 에너지
행 3: 산업재 / 유틸리티 / 소재
행 4: 커뮤니케이션 / 부동산 / [전체 보기]
```

```
┌─ SECTOR HUB ──────────────────────────────────────────────┐
│                                                            │
│   어떤 산업을 분석하시겠습니까?                             │
│                                                            │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐     │
│  │{섹터명1} │ │{섹터명2} │ │{섹터명3} │ │{섹터명4} │     │
│  │{tier_1}  │ │{tier_2}  │ │{tier_3}  │ │{tier_4}  │     │
│  │{n1}개 종목│ │{n2}개 종목│ │{n3}개 종목│ │{n4}개 종목│    │
│  └──────────┘ └──────────┘ └──────────┘ └──────────┘     │
│       ...  (11개 섹터 전체를 동일 형식으로 반복)            │
│                                                            │
│  섹터 티어 = 섹터 평균 총점 기준 (런타임 계산)              │
└────────────────────────────────────────────────────────────┘
```

**섹터 카드 자동 생성 규칙:**
```
섹터 티어   : 섹터 내 전체 종목 평균 총점 → 티어 변환
종목 수     : EDGAR에서 해당 섹터 10-K 제출 기업 수
카드 색상   : 섹터 티어 컬러 (S=#22C55E, A=#60A5FA, B=#F59E0B, C=#EF4444)
```

### 5.3 섹터 스탯 보드 (섹터 허브 → 섹터 카드 클릭 후 진입)

### 5.5 지표 설명 팝업 (ⓘ 클릭 또는 셀 hover)

> 뇌동매매 방지의 핵심 장치.
> 모든 지표에 팝업이 연결되어 있어 숫자가 나오면 즉시 이해할 수 있도록 한다.

```

┌─ {indicator_fullname} ({indicator_name}) ── [{tier}티어] ──┐
│                                                           │
│  📖 무엇인가요?                                           │
│  {indicator_definition}  ← 섹션 6.1 자동 선택            │
│                                                           │
│  🏭 {섹터명} 섹터에서 왜 중요한가요?                      │
│  {sector_x_indicator_reason}  ← 섹션 6.3 자동 선택       │
│                                                           │
│  📊 이 종목: {indicator_name} {value}  →  [{tier}티어]    │
│     섹터 정상 범위: {lo}~{hi}  /  섹터 평균: {avg}        │
│     섹터 내 백분위: 상위 {percentile}%                    │
│                                                           │
│  ⚠️ 알아두세요 (해당 지표에 해석 함정이 있는 경우만 표시)  │
│  {caution_text}                                           │
│                                                           │
└───────────────────────────────────────────────────────────┘
```

**팝업 자동 생성 규칙:**
```
모든 지표에 대해 다음 4개 항목을 자동 조합:
  1. 정의 + 예시 (지표 사전 참조 — 섹션 6)
  2. 섹터별 중요도 설명 (섹터×지표 매핑 참조 — 섹션 6)
  3. 현재 수치 + 티어 + 섹터 정상 범위 + 섹터 평균 + 섹터 내 백분위
  4. 주의사항 (해당 지표에 알려진 해석 함정이 있는 경우)
```

## 6. 교육 콘텐츠 사전

### 6.1 지표 정의 사전

```
GPM (매출총이익률):
  정의: "매출 100원 중 원가를 빼고 남은 금액의 비율입니다."
  예시: "GPM 40% = 100원에 팔면 원가 60원, 40원이 남음"

OPM (영업이익률):
  정의: "매출 100원 중 영업으로 남긴 이익의 비율입니다."
  예시: "OPM 20% = 100원 팔면 20원이 영업이익으로 남음"

NPM (순이익률):
  정의: "세금·이자까지 다 내고 최종으로 남는 이익의 비율입니다."
  예시: "NPM 15% = 100원 팔면 최종 15원이 남음"

ROE:
  정의: "주주가 맡긴 돈 100원으로 얼마를 벌었는지 보는 지표입니다."
  예시: "ROE 20% = 100원 투자 → 20원 순이익"

ROA:
  정의: "회사가 보유한 자산 전체로 얼마를 벌었는지 보는 지표입니다."
  예시: "ROA 10% = 자산 100원 → 10원 순이익"

매출 YoY:
  정의: "작년보다 매출이 얼마나 늘었는지를 나타냅니다."
  예시: "YoY 10% = 작년 100억 → 올해 110억"

매출 3Y CAGR:
  정의: "3년간 매출이 연평균 얼마씩 성장했는지 나타냅니다."
  예시: "3Y CAGR 15% = 3년 전 100억 → 현재 152억"

DR (부채비율):
  정의: "자기 돈 대비 빌린 돈이 얼마나 되는지 보는 지표입니다. 낮을수록 안전합니다."
  예시: "DR 50% = 내 돈 100원에 빚이 50원"

CR (유동비율):
  정의: "1년 안에 갚아야 할 빚을 갚을 능력이 있는지 보는 지표입니다."
  예시: "CR 150% = 단기 빚 100원을 갚고도 50원이 남음"

ICR (이자보상배율):
  정의: "영업이익으로 이자를 몇 번이나 갚을 수 있는지 보는 지표입니다."
  예시: "ICR 5배 = 이자 100원을 영업이익으로 5번 낼 수 있음"

총자산회전율:
  정의: "보유 자산 1원당 얼마의 매출을 만들어내는지 보는 지표입니다."
  예시: "자산회전율 0.8 = 자산 100원 → 매출 80원"

PER:
  정의: "현재 주가가 연간 순이익의 몇 배인지 나타냅니다."
  예시: "PER 20배 = 지금 투자하면 순이익으로 20년 후 본전"

PBR:
  정의: "주가가 순자산의 몇 배인지 나타냅니다. 1배 미만이면 장부가 이하."
  예시: "PBR 2배 = 순자산 100원짜리 회사를 200원에 사는 것"

PSR:
  정의: "주가가 매출의 몇 배인지 나타냅니다. 이익이 없는 성장주 평가에 사용."
  예시: "PSR 3배 = 연 매출 100원짜리 회사를 300원에 사는 것"

EV/EBITDA:
  정의: "기업 전체 가치가 영업이익+감가상각의 몇 배인지 나타냅니다."
  예시: "EV/EBITDA 10배 = 기업 가치 회수에 10년 소요 (단순 계산)"

FCF (잉여현금흐름):
  정의: "영업으로 벌어들인 현금에서 설비 투자를 빼고 실제 남는 돈입니다."
  예시: "FCF 1조 = 이익을 내고 투자도 하고도 1조가 남음"

FCF Margin:
  정의: "매출 중 실제로 회사에 남는 현금의 비율입니다."
  예시: "FCF Margin 15% = 100원 매출 → 15원 현금이 남음"

배당수익률:
  정의: "주가 대비 배당금의 비율입니다. 주식의 '이자율'과 같습니다."
  예시: "배당수익률 3% = 100만원 주식 → 연 3만원 배당"
```

## 7. 인사이트 자동 생성 규칙

### 7.1 섹터 스탯 보드 인사이트

```
Step 1. 섹터 티어 분포 문장
  S_pct = S티어 종목 수 / 전체 × 100

  S_pct >= 30% → "{섹터} 섹터는 전반적으로 강한 상태입니다.
                   {N}개 기업 중 {s}개가 섹터 최상위권(S티어)입니다."

  S_pct >= 15% → "일부 S티어 기업이 {섹터} 섹터를 이끌고 있습니다."

  C티어 >= 30% → "{섹터} 섹터 내 {c}개 기업이 C티어입니다.
                   섹터 전반에 주의가 필요한 상황입니다."

Step 2. 핵심 지표 1번 상태 문장
  A/S → "특히 {지표명}이 섹터 강점 — 평균 {avg} (정상 범위 {lo}~{hi})"
  C   → "{지표명} 섹터 평균 {avg}는 정상 범위 하단. 주의 구간."
```

> "뇌동매매 하지 말고, 알고 투자하자" 컨셉을 가장 직접적으로 표현하는 기능.

```
┌─ 오늘의 종목 ──────────────────────────────────────────────┐

│                                                            │
│  PER {n}배는 {섹터} 섹터 평균({avg}배)보다 낮은 수준입니다.│
│  ROE {roe}%는 섹터 내 상위 {pct}%입니다.                  │
│                                                            │

│                                                            │
│  → [자세히 보기]                                           │

```

**오늘의 종목 선정 규칙:**
```python
# 조건 1: Overall A티어 이상
# 조건 2: 전일 대비 주가 하락 (price_change_pct < 0)
# 조건 3: 밸류에이션 B티어 이상 (과도한 고평가 제외)
# 조건 4: 위 조건 충족 종목 중 총 투자 점수 상위 1개 선정
# 없으면: Overall A티어 이상 종목 중 랜덤 1개

def pick_today_stock(db_conn, date: str) -> str:
    # ※ tier_valuation은 tiers 테이블의 스탯별 대표 티어 컬럼 (Core §13 참조)
    # ※ scores 테이블에는 overall_tier만 있음 — 스탯별 티어는 반드시 tiers 테이블 JOIN
    candidates = db_conn.execute("""
        SELECT c.ticker, sc.total_score, p.price_change_pct
        FROM companies c
        JOIN scores  sc ON c.ticker = sc.ticker
        JOIN tiers   t  ON sc.ticker = t.ticker AND sc.fiscal_year = t.fiscal_year
        JOIN prices  p  ON c.ticker = p.ticker AND p.price_date = ?
        WHERE sc.overall_tier IN ('S', 'A')
          AND t.tier_valuation IN ('S', 'A', 'B')
          AND p.price_change_pct < 0
        ORDER BY sc.total_score DESC
        LIMIT 1
    """, (date,)).fetchone()
    return candidates[0] if candidates else None
```
